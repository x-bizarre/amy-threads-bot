# SETUP — как запустить Amy с нуля

Гайд для тех, кто берёт репозиторий и поднимает бота под свой бренд. Идём сверху
вниз. Каждый шаг — отдельный сервис; в конце связываем всё на Vercel.

> Время на всё: ~1–2 часа, если ничего из сервисов раньше не настраивал(а).
> Ничего сложного, просто много мелких ключей. Заводи их по очереди и сразу
> вписывай в `.env` (скопируй его из `.env.example`).

## 1. Подготовка проекта

```bash
git clone <твой-форк>.git
cd amy-threads-bot
npm install
cp .env.example .env       # сюда будешь вписывать ключи по ходу гайда
```

Проверь, что код собирается:
```bash
npm run typecheck
```

---

## 2. Бренд — кто ты

Открой `.env` и заполни первый блок:
```
BRAND_NAME="Название твоего продукта"
BRAND_SITE="example.com"
BRAND_FOUNDER=""            # имя для постов от первого лица, можно пусто
```
Потом загляни в папку [`prompts/`](prompts/) и заполни голос бренда — это важнее
любого ключа. Начни с [`prompts/README.md`](prompts/README.md).

## 3. Threads API

Threads нужен, чтобы публиковать посты и читать комментарии.

1. Зайди на <https://developers.facebook.com/> → **My Apps** → **Create App**.
2. Тип приложения — выбери с доступом к **Threads API** (Use case → Threads).
3. В настройках приложения подключи продукт **Threads** и добавь разрешения:
   `threads_basic`, `threads_content_publish`, `threads_manage_replies`,
   `threads_read_replies`.
4. Добавь свой Threads-аккаунт как тестового пользователя (если приложение в
   режиме разработки).
5. Сгенерируй **long-lived access token** (живёт ~60 дней).
6. Узнай свой **числовой user id** — подставь токен в этот адрес и открой в
   браузере (в ответе будет поле `id`):
   ```
   https://graph.threads.net/v1.0/me?fields=id,username&access_token=ТВОЙ_ТОКЕН
   ```

Впиши в `.env`:
```
THREADS_ACCESS_TOKEN="EAAB...твой токен"
THREADS_USER_ID="1784xxxxxxxxxxx"
THREADS_USERNAME="твой-хэндл-без-собаки"
```

> Токен живёт 60 дней, но бот умеет обновлять его сам — см.
> [раздел 11](#11-обслуживание). Первый токен всё равно заводишь руками.

## 4. Telegram-бот

Через него ты одобряешь посты и ответы и получаешь алерты.

1. Напиши [@BotFather](https://t.me/BotFather) → `/newbot` → получи
   **TELEGRAM_BOT_TOKEN**.
2. Узнай свой **chat id**: напиши что-нибудь своему боту, потом открой
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates` — в ответе найди `chat.id`.

```
TELEGRAM_BOT_TOKEN="123456:ABC..."
TELEGRAM_CHAT_ID="твой_числовой_id"
```

Webhook (чтобы бот реагировал на кнопки) подключается уже после деплоя — см.
[раздел 10](#10-проверка-что-всё-живо).

## 5. OpenRouter (LLM)

Бот пишет посты и ответы через модель на OpenRouter.

1. Зарегистрируйся на <https://openrouter.ai/>, пополни баланс (хватит пары
   долларов на старт).
2. Создай API-ключ.

```
OPENROUTER_API_KEY="sk-or-..."
```

Какая модель используется — задаётся в коде (`lib/openrouter.ts` и
`lib/post-generator.ts`, поле `model`). По умолчанию стоит Claude Sonnet —
можешь поменять на любую из каталога OpenRouter.

## 6. GitHub PAT (хранилище очереди)

Бот хранит очередь постов и состояние прямо в твоём репозитории (на Vercel
файловую систему писать нельзя, поэтому пишем коммиты через GitHub API).

1. GitHub → **Settings** → **Developer settings** →
   **Personal access tokens** → **Fine-grained tokens** → **Generate**.
2. Доступ — только к этому репозиторию.
3. Разрешения: **Contents → Read and write**.

```
GITHUB_TOKEN="github_pat_..."
GITHUB_REPO_OWNER="твой-логин-или-орг"
GITHUB_REPO_NAME="amy-threads-bot"
GITHUB_BRANCH="main"
```

## 7. Upstash Redis

Redis хранит состояние: какие комменты уже видели, активные диалоги, свежий
Threads-токен, журнал решений, защита от двойной публикации.

1. Зарегистрируйся на <https://upstash.com/> → **Create Database** (Redis).
2. Включи **REST API** и скопируй URL и токен.

```
UPSTASH_REDIS_REST_URL="https://...upstash.io"
UPSTASH_REDIS_REST_TOKEN="..."
```

## 8. Деплой на Vercel

1. Зайди на <https://vercel.com/>, импортируй свой GitHub-репозиторий.
2. Framework — Next.js определится сам.
3. В **Settings → Environment Variables** перенеси ВСЕ переменные из `.env`.
4. Задеплой.

`CRON_SECRET` Vercel создаст сам, когда включишь cron (следующий шаг).

## 9. Расписание (cron)

Расписание уже описано в `vercel.json` (автопостинг, обновление токена,
недельный анализ). Vercel подхватит его при деплое.

Часть задач (слежение за комментами каждые 5 минут, публикация одобренных
ответов) вынесена в **GitHub Actions** — потому что бесплатный тариф Vercel
разрешает cron только раз в день. Эти workflow лежат в `.github/workflows/`.
Чтобы они работали, добавь в **Settings → Secrets → Actions** репозитория:
- `VERCEL_URL` — адрес твоего деплоя (например `https://amy-xxx.vercel.app`)
- `CRON_SECRET` — то же значение, что в Vercel

> Почему так, а не отдельный сервер: чтобы тебе не платить за хостинг и не
> поднимать инфраструктуру. Для одного бренда этой связки достаточно. Если
> когда-нибудь упрёшься в лимиты — вынесешь планировщик отдельно, но для старта
> не нужно.

> ⚠️ На **бесплатном тарифе Vercel** cron срабатывает редко (около раза в сутки),
> поэтому несколько слотов автопостинга в день могут отрабатывать не все. Если
> важно публиковать чаще — либо подними тариф, либо запускай автопост через
> GitHub Actions (как уже сделано для слежения за комментами).

## 10. Проверка, что всё живо

1. **Подключи Telegram webhook.** Это обязательный шаг — без него кнопки
   одобрения работать не будут. Когда деплой готов и его адрес известен, просто
   открой в браузере:
   ```
   https://<адрес-твоего-деплоя>/api/bot/telegram-webhook?setup=1
   ```
   Страница ответит «Webhook подключён». После этого бот начнёт получать твои
   сообщения и нажатия кнопок.
2. **Напиши боту `/help`** в Telegram — он должен ответить списком команд. Если
   ответил — связка работает.
3. **Положи первый пост** в `queue/` (формат — `queue/README.md`), запушь.
4. **Запусти автопост вручную** (не дожидаясь расписания): GitHub → вкладка
   **Actions** → workflow **«Autopost (manual trigger)»** → кнопка **Run
   workflow**. Этот способ сам отправляет нужный секрет.
   > ⚠️ Не пытайся открыть `/api/cron/autopost` в браузере — там стоит защита,
   > и без секретного заголовка вернётся ошибка 401. Запускай только через Actions.

## 11. Обслуживание

- **Токен Threads** обновляется сам: cron `token-refresh` раз в неделю проверяет
  срок и, если осталось меньше 10 дней, обновляет токен и пишет тебе в Telegram.
  Если обновление не удалось — придёт алерт, и тогда заведи новый токен руками
  (раздел 3) и обнови `THREADS_ACCESS_TOKEN` в окружении деплоя.
  Проверить вручную можно так (подставь свой адрес и `CRON_SECRET`):
  ```bash
  curl -H "Authorization: Bearer ТВОЙ_CRON_SECRET" \
    "https://<адрес-деплоя>/api/cron/token-refresh?force=1"
  ```
- **Зимнее/летнее время.** Расписание в `vercel.json` — в UTC. При переходе на
  другое время сдвинь часы, если хочешь публиковать в те же местные часы.
- **Правишь промпты** в `prompts/*.md` → коммить и пушь → бот подхватит за минуту.
- **Что-то сломалось?** Смотри логи в Vercel → Functions, и журнал решений бота
  (он пишется в Redis-список `decisions:log`).
