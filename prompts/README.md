# prompts/ — мозг бота, который ты правишь руками

Это самая важная папка. Здесь живёт «личность» твоего бота — голос, продукт,
кастдев-вопросы. **Бот читает эти файлы напрямую** при каждом запуске, поэтому
тебе не нужно лезть в код: правишь markdown — меняется поведение.

| Файл | Что задаёт | Кто читает в коде |
|---|---|---|
| [`brand-voice.md`](brand-voice.md) | голос бренда, продукт, красные линии | `lib/brand.ts` → ответы и посты |
| [`castdev.md`](castdev.md) | что выясняем у аудитории | `lib/castdev.ts` → discovery-посты |
| [`content-strategy.md`](content-strategy.md) | типы постов и их структура | справочник (числа — в `lib/content-strategy.ts`) |
| [`audience-icp.md`](audience-icp.md) | портрет твоего клиента | держи под рукой, заполняя остальные |

## Как работают подстановки

В тексте можно писать `{{BRAND_NAME}}`, `{{BRAND_SITE}}`, `{{BRAND_HANDLE}}`,
`{{BRAND_FOUNDER}}` — бот сам подставит значения из `lib/brand-config.ts` (или
`.env`). Эти плейсхолдеры не трогай, остальной текст — твой.

## Порядок заполнения

1. Сначала `audience-icp.md` — поймёшь, для кого пишешь.
2. Потом `brand-voice.md` — самый важный: как звучит бренд.
3. Потом `castdev.md` — что хочешь узнавать у людей.
4. `content-strategy.md` — при желании подкрути типы постов.

Значки 👉 в файлах помечают места, которые обязательно перепиши под свой продукт.
Не нужно «идеально» — заполни как умеешь, бот заработает, дальше поправишь.

## Важно про деплой

Поскольку бот читает эти файлы из репозитория через GitHub API (на Vercel
файловая система только для чтения), после правки промптов **закоммить и
запушь** изменения — бот подхватит их в течение минуты.
