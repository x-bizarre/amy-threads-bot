# Amy — бот для Threads (автопостинг + аутрич + кастдев)

Amy — это бот, который ведёт аккаунт бренда в Threads почти сам:

1. **Автопостинг** — берёт посты из очереди (`queue/*.md`) и публикует по расписанию.
2. **Ответы на комментарии** — следит за комментами под постами, готовит черновик
   ответа и присылает тебе в Telegram. Публикует только после твоего ✅.
3. **Аутрич** — находит чужие релевантные посты, пишет к ним тёплый комментарий
   (тоже с твоим одобрением).
4. **Пассивный кастдев** — почти каждый ответ заканчивается живым вопросом, а раз
   в неделю Amy анализирует все собранные комменты и достаёт из них инсайты про
   твою аудиторию.

Ты остаёшься «в петле»: бот предлагает — ты одобряешь кнопкой. Ничего не уходит
в публикацию без тебя.

> Это **обезличенный шаблон**. Здесь нет ни одного реального поста, токена или
> названия бренда — только код и пустые шаблоны, которые ты заполняешь под себя.

---

## Как это устроено

- **Next.js + TypeScript**, крутится на **Vercel** (cron + serverless-ручки).
- **Очередь постов** — markdown-файлы прямо в репо (`queue/`).
- **Управление** — через Telegram-бота (одобряешь/правишь посты и ответы).
- **LLM** — генерация и переводы через **OpenRouter** (модель настраивается).
- **Состояние** диалогов/аутрича — в **Redis** (например, Upstash).

```
prompts/                ← ГОЛОС БОТА: голос бренда, кастдев, стратегия (правишь markdown)
lib/brand-config.ts     ← кто ты: имя бренда, сайт, хэндл  (СТАРТУЙ ОТСЮДА)
queue/                  ← очередь постов (тут лежат 2 примера-рыбы)
app/api/cron/           ← ручки: autopost, replies-watcher, token-refresh, …
app/api/bot/            ← Telegram-вебхук (команды и кнопки)
lib/                    ← логика: Threads, Telegram, OpenRouter, Redis, GitHub
```

Главная идея: вся «личность» бота вынесена в **`prompts/*.md`** — бот читает их
напрямую. Тебе НЕ нужно лезть в код, чтобы поменять голос: правишь markdown,
коммитишь, пушишь — бот подхватывает за минуту.

## Быстрый старт

1. **Установи зависимости**
   ```bash
   npm install
   ```

2. **Настрой бренд.** Скопируй `.env.example` → `.env`, заполни `BRAND_NAME`,
   `BRAND_SITE`, `THREADS_USERNAME`.

3. **Задай голос бота** в [`prompts/`](prompts/) — это и есть твоя «настройка под
   себя». Начни с [`prompts/README.md`](prompts/README.md). Код возьмёт текст
   оттуда сам — переносить ничего не надо.

4. **Положи свои посты** в `queue/` (формат — в [`queue/README.md`](queue/README.md))
   и удали два примера-рыбы.

5. **Проверь, что собирается**
   ```bash
   npm run check     # typecheck + lint
   ```

6. **Полный деплой** — пошагово по каждому сервису в **[SETUP.md](SETUP.md)**
   (Threads app, Telegram, OpenRouter, GitHub PAT, Upstash, Vercel).

## Что нужно подключить (ключи)

Все ключи — в `.env` (см. `.env.example`). Ничего из этого в репозиторий не
попадает.

| Сервис | Зачем |
|---|---|
| Threads API | публикация постов и ответов |
| Telegram Bot | одобрение/правка постов и ответов, алерты |
| OpenRouter | генерация постов и переводы |
| GitHub PAT | бот хранит очередь как файлы в репо |
| Redis (Upstash) | состояние диалогов и аутрича |

## Команды в Telegram

| Команда | Что делает |
|---|---|
| `/start`, `/help` | приветствие и список команд |
| `/generate` (`/gen`) | сгенерить новый пост на одобрение |
| `/next` | показать следующий пост из очереди |
| `/preview` | предпросмотр того, что опубликуется ближайшим |
| `/edit` | поправить черновик текстом |
| `/outreach` | запустить поиск чужих постов для комментариев |
| `/analyze` | прогнать недельный анализ комментов (кастдев) |
| `/register <ссылка>` | начать следить за комментами под конкретным постом |

## Расписание (cron)

Настраивается в `vercel.json` и в `.github/workflows/` (для тех ручек, что
на бесплатном тарифе Vercel cron'ом не вытянуть). Все времена — в UTC, не
забудь сдвинуть при переходе на зимнее/летнее время.

| Ручка | Когда | Что |
|---|---|---|
| `autopost` | несколько раз в день | публикует пост из очереди |
| `queue-refill` | ежедневно | догенерирует посты, если очередь пустеет |
| `replies-watcher` | каждые ~5 мин | ищет новые комменты |
| `publish-pending` | каждые ~5 мин | публикует одобренные ответы |
| `weekly-analysis` | раз в неделю | кастдев-анализ комментов |
| `token-refresh` | раз в неделю | сам обновляет Threads-токен, не даёт боту умереть |

## Что встроено для надёжности

- **Авто-обновление Threads-токена.** Токен живёт 60 дней; бот обновляет его сам
  и шлёт алерт в Telegram, если что-то не так. Не нужно помнить про продление.
- **Защита от двойной публикации.** Лок + пометка в Redis — даже если два cron
  совпали или запись статуса сорвалась, пост не уйдёт дважды.
- **Человек в петле.** Ни один ответ и ни один аутрич-комментарий не публикуется
  без твоего ✅ в Telegram. Это и есть главная защита от «бот сказал глупость».
- **Журнал решений.** Бот пишет, почему ответил/пропустил — удобно дебажить.
- **CI.** На каждый push прогоняется typecheck + lint (`.github/workflows/ci.yml`),
  чтобы ты сразу видел(а), если правка сломала код.
